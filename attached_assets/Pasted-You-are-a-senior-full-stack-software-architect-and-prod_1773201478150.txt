You are a senior full-stack software architect and product designer.

Your task is to design and generate the first version (MVP) of a modern web application for construction architects that helps them manage their projects, track construction site visits, store documents, and visualize all their projects on an interactive geographic map.

The goal is to create a professional SaaS platform for building architects that combines:

Project management

Site visit tracking

Geographic visualization of projects

Document management

Reporting tools

This application should solve real problems architects face when managing multiple construction sites.

PRODUCT VISION

Architects usually manage many construction projects simultaneously.
Their information is often scattered across emails, WhatsApp messages, photos, and documents.

This platform centralizes everything into one intelligent system.

The core innovation is a smart interactive map of projects and visits similar to a Snapchat-style map, where architects can visually explore all their construction sites and visits geographically.

TARGET USERS

Primary users:

Construction architects who manage:

residential projects

buildings

renovations

construction supervision

They need tools to:

track site visits

document progress

manage clients and contractors

store plans and documents

visualize their projects geographically

CORE FEATURES (MVP)
1 Project Management

Architects can create projects including:

Project name

Client name

Location (GPS coordinates)

Project type

Budget

Status (planned / active / completed)

Description

Each project should include:

timeline

documents

photos

visits history

collaborators

2 Interactive Project Map (Core Innovation)

Create a main map interface showing all projects.

Each project is represented by a map pin.

Clicking a pin should open a project preview containing:

project name

photos

number of visits

project status

last visit date

Map features:

zoom

filter projects by status

cluster nearby projects

highlight active projects

Use a mapping library such as:

Leaflet or Mapbox.

3 Site Visit Tracking

Architects frequently visit construction sites.

Allow them to record visits with:

date

notes

photos

detected issues

progress status

Visits should be linked to a project.

Each visit creates a visit report entry.

4 Visit Timeline

Inside each project page, show a timeline of visits including:

visit date

notes

photos

progress updates

This allows architects to track the evolution of a construction site.

5 Document Management

Allow uploading and storing files such as:

architectural plans

PDFs

contracts

images

Files should be linked to specific projects.

6 Dashboard

Create a dashboard that shows:

total projects

active projects

upcoming visits

recent activity

project distribution on the map
